# Limelight yardstick metrics features

The entire focus of the site should be around the taxonomic rating.

A user's profile should list each movie that they have rated with the rating. They're given grouping by by score and then taxonomic rating.

A movie's profile should list each unique taxonomic rating it's received paired with the percentage of reviews for that movie with that rating, highest percentage first.

Recommendations should be a community style experience.  We need to develop some sort of commonality analysis looking from user profile to user profile to find common tastes to users. Reviewed a bunch of the same movies. Bunch the same ways or close to so my particular tastes most readily aligned with these other particular users. These other users have rated movies that I haven't rated yet, so the site recommends those movies. I can then either add those movies to a watch list or rate them. Adding new readings updates my commonalities.

However, I want to avoid biasing users' ratings, so they should not be able to see specific rating data of any movie that they themselves have not yet rated. They can see the movie metadata: title, poster, description, content rating, etc; but no rating data.

I definitely don't want to use the texanimic rating itself as a filter or search feature. It's purely an analytical tool to analyze fewer taste and compare them with other viewers.

At some point, who want to develop some sort of analytics to take a broader view across all users to try to identify clusters groupings of various tastes.
